用Hexo搭建GitHub博客

發表於 2016-06-01   |   分類於 Tech   |  

前言

會折騰用hexo搭建github博客純粹是基友安利。然而本人是技術小白完全沒有程式背景，並且人品不佳，因此基本上大家遇上的很多問題都遇上了，這之後再開另一篇總結一下，裡面提到的問題五花八門且技術質量不高，也請多多指教。本次的架站原本是在windows系統下架站成功，然後事後又遇上了些問題而無法解決的更新問題，所以改搬到了Mac底下來架設，因此以下的文章是在Mac OSX系統底下陳述。

目標

主要一般是自我生活的分享和隨筆搭配著折騰的技術，簡單說以分享自已的生活為主技術為輔的博客，因此技術的交流可能比較使不上力，但還是請大家多多指教。

教程和參考

  • HEXO官方網站教程

    可以補足網路個人版教程簡略的部分，或者是除錯找資源的好地方，官方主題模板好查詢的地方。

  • GitHub Wikis

    寫得很詳盡，不過是英文版本，可以當作有問題查閱的資料庫。

  • 教程1:DoubleDan’s blog的教程

    主幹的安裝是照著這個教程步驟安裝的，但掛上自己域名的部分可以最後再看。不過寫的比較重點，所以詳細的部分可以參看下面的教程。

  • 教程2:Code_bunny的教程（有三篇）

    截圖很多，解說十分詳盡，因此後面改細節的部分，大量參閱這教程。

安裝上的問題

安裝Git

(由於GitHub是以工具Git進行的版本控制，專門來存放內容的虛擬主機服務)

  1. 照著教程1寫的安裝，如果沒有安裝過的才需要安裝MacXcode，根據教程來說是為了裝Homebrew，然後執行列的終端機Terminal可以在應用程式裡找到。

  2. 去官網手動安裝 照著上面寫的裝上。

安裝Node.Js

(Node.Js是通過JavaScript和一系列模組來編寫伺服器端應用。)

教程1上說的先安裝nvm(Node Version Manager)主要是為了管理node的版本，而詳盡的部分可以觀看icaru4’s blog，安裝完再回到前面教程裝Node.Js。

配置SSH Keys

(SSH Keys是SSH文件傳輸加密登陸的便捷方法。)

照著前面教程1使用就可以在Mac底下生成SSH Key。然後要找到id_rsa.pub的位置的話，如沒看到話就在指令列找尋拷貝再貼上GitHub。

$ cd .ssh

$ sudo vim id_rsa.pub

(選擇read only後就會看到SSH Key，由於用Mac可以直接滑鼠複製)

外插：若是用windows介面下則必須多裝上ssh-agent，整個過程可以參看GitHub Help裡所寫的

安裝Hexo

照著前面的教程所說安裝但由於Hexo版本升級到3.0後第一步指令改成以下

$ npm install hexo-cli -g

剩下的部分可以照著教程安裝完成。

若想要確定自己的Hexo版本：在剛剛建立的文件夾底下輸入

$ hexo version

可以看到回車

hexo: 3.2.0

hexo-cli: 1.0.1

部署Hexo到GitHub上

打開剛剛建立的文件夾底下的_config.yml在此設定，根據教程2博客(二)的設定改成自己所想要的設定。但是注意教程最底下的deploy有誤，漏寫了.git

repository: https://github.com/你的名稱/你的名稱.github.io.git

在這頁文件上：後面要空一格，不然回車會出現以下結果

FATAL can not read a block mapping entry; a multiline key may not be an implicit key at line 哪行, column哪列

改完後再執行

$ hexo d

正常狀況照著教程裝上插件後就可以搞定，然而偶爾也會遇到回車出現：

fatal: Not a git repository(or any of the parent directories): .git

代表git倉庫的遺失可以選擇.deploy_git文件夾或者是

$ cd .deploy_git

$ git init

這也可以解決。

更換主題

照著教程2博客(三)來選擇拷貝主題，本人基本上是選擇官網上主題的NexT其官方的使用說明寫的十分清楚。

  • 設置頭像：若是手機拍攝的照片有時候照片會轉個90度的問題，儘管照片看來是正確的，這其實是照片本身的問題，所以最後用照片後處理軟件重新另存一次就可以解決。

  • 社交連結：微博照著上面所寫的話會連結不到，改以微博: http://weibo.com/u/數字帳號就可以成功連上。

購買域名

基本上是照著教程1所做的，然而DNS A紀錄的ip地址要十分注意是GitHub站的，可以用ping來查看

$ ping Github帳號.github.io

所得到的ip填上，基本上這樣就可以搞定。然而有些教程會教用若是CNAME可以參看此博客注意CNAME沒後綴且要放在網站目錄底下才不會每次生成被刪除。

結語

以上雖然很碎片，但基本上遇到的問題都記錄了。大家能夠閱讀十分感激，同時也感謝了這些教程的幫忙。請大家多多指教。
